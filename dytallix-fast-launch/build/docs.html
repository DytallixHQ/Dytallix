Introduction

Welcome to the Dytallix developer documentation. The platform is built around post-quantum cryptography (PQC) with NIST-approved primitives including Kyber, Dilithium, SPHINCS+, and Falcon. Every guide is opinionated toward real-world workflows so your team can move from prototype to production quickly and safely.

Key advantages
  • Post-quantum secure by default across wallets, APIs, and contracts.
  • Hybrid classical + PQC signatures with automated key rotations.
  • Comprehensive testnet and observability tooling out of the box.
Build in minutes

Quick Start

Install the SDK, create a wallet, and submit your first transaction.

# Install via npm
npm install @dytallix/sdk

# Or with yarn
yarn add @dytallix/sdk
import { DytallixClient } from '@dytallix/sdk';

const client = new DytallixClient({
  endpoint: 'https://testnet.dytallix.com',
  apiKey: process.env.DYTALLIX_API_KEY,
});

const wallet = await client.createWallet({
  algorithm: 'dilithium',
  securityLevel: 3,
});

console.log('Address:', wallet.address);
Next up: Fund your wallet from the testnet faucet and explore live transactions inside the explorer.
Core concepts

What you need to know

Understanding these pillars will make the rest of the documentation intuitive.

Wallet management

Hardware-backed secrets, secure enclaves, and encrypted local storage power every wallet flow.

  • Generate Kyber and Dilithium keypairs.
  • Enable hybrid RSA + PQC signatures.
  • Rotate keys with automated revocation.

Transactions

All transactions are signed using PQC primitives with telemetry streamed to the dashboard.

  • Gas fees and execution visibility via explorer.
  • Replay protection and nonce management.
  • Webhooks and GraphQL subscriptions.

Smart contracts

Deploy WASM contracts with PQC verification gates and rollback plans.

  • Environment diffing before deploy.
  • Governance approvals baked in.
  • Security attestations tracked automatically.

Observability

Dashboards surface health metrics, rate limits, and PQC integrity in real time.

  • Usage analytics with anomaly detection.
  • Alerting integrations for Slack and PagerDuty.
  • Historical exports for compliance and audits.
API reference

REST API

Authenticate with your API key using the Authorization: Bearer <key> header.

POST /api/v1/wallet/create

Create a new PQC wallet for a given algorithm and security level.

GET /api/v1/explorer/block/:height

Retrieve block data with signature insights and transaction counts.

POST /api/v1/contracts/deploy

Deploy a smart contract with environment diffing and PQC verification.

Real-time

GraphQL API

The GraphQL endpoint lives at https://api.dytallix.com/graphql.

query LatestBlocks {
  blocks(limit: 10, orderBy: HEIGHT_DESC) {
    height
    timestamp
    transactions {
      hash
      status
      signatureAlgorithm
    }
  }
}
Tip: Use subscriptions for real-time block feeds and pipeline them into your monitoring or analytics stack.
